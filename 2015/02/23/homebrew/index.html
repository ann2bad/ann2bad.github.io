      简洁优雅更有逼格的为 Mac OS X 安装软件

      Post on Feb 23 2015

      记得刚用上Mac OS的时候，感觉mac下软件的安装简直是太无敌了。告别无脑“下一步”，也不用担心被莫名其妙改了浏览器首页，更不会“买一送一”的顺便给你装一些猜你喜欢的流氓软件。每次装个新东西都要提高警惕，小心翼翼，所以，赶快和windows说byebye吧，窗外的世界更精彩！

      但是，愈深入的折腾mac，愈发现也不想象中那么完美。各种软件的安装流程不尽一致，这让安装及管理各种软件的过程非常痛苦。（主要原因是Unix Like系统的框架结构，以及Apple将IOS上SandBox理念硬性迁移到AppStore上的zuo&die的问题）。

      各种各样的安装体验，让人眼花缭乱

      Mac OS X下安装软件的标准流程是挂载一个.dmg的镜像文件，然后把App的图标拖入到“应用程序”文件夹下。遗憾的是，大部分程序并不一定遵守这个君子之约，各自发明创造。

      “你懂的”类型

      好的体验应该是.dmg镜像挂载打开后，要将App的bundle文件指向应用程序文件夹的Softlink图标上，示意用户要用鼠标拖进去。然而例如Sublime Text 2，其假定用户完全知道其中的奥妙，白白的背景下，没有任何提示。两个光秃秃的图标共处一室，这究竟是要闹哪样？你懂的！

      “没有然后”类型

      对一个新用户来说，下载后是.zip压缩包的软件，都有一个共同的德行，解开zip后你就得到了一个App，恭喜！然后，就没有然后了。哥只能帮你到这里，接下来这玩意爱放哪就放哪吧。这类软件比如Evernote的Skitch。

      “双击安装”类型

      有些App的.dmg里只有一个貌似App图标的东东，而应用程序文件夹缺无情的消失了。你若自以为NB的自作主张将其放到应用程序里，结果你就SB了。这玩意居然是让你双击-下一步…-完成来安装！你是windows请来的救兵吗？

      “IOS”类型

      这个就简单了，进到App Store，找到想要的APP，付款-下载，搞定！但是。。。由于Apple强制App Store中的应用必须在Sandbox中，不许越雷池一步，结果很多高大上的软件都坚决抵制App Store了，现在看看App Store上残留的App比百度搜索结果强不到哪去。

      2. 安装homebrew (一行命令直接搞定，复制，粘贴，回车)
      3. 安装homebrew-cask (同上)
